# 📊 FDA-10: Multimodal Credit Risk Analysis

## 🔍 Overview

FDA-10 is a **large-scale, end-to-end multimodal credit risk analytics pipeline** that integrates structured financial data from **SEC XBRL filings** with **NLP-derived textual features** to predict corporate credit ratings and investment-grade classification.

The project demonstrates how combining **quantitative financial ratios** with **qualitative signals** such as sentiment, risk, and uncertainty improves traditional credit risk models. The entire workflow is designed to be **scalable, reproducible, and configurable**, and ships with an interactive **Streamlit dashboard** that serves live predictions from the trained models.

🔗 **Live Dashboard:** https://multimodal-credit-risk-analysis.streamlit.app/

---

## 🎯 Key Objectives

- ⚙️ Process **large-scale SEC XBRL financial data** across multiple years
- 🔧 Engineer financially meaningful ratios for credit risk assessment
- 📝 Extract NLP-based indicators from MD&A-style textual data
- 🤖 Build and compare **financial-only vs multimodal** ML models
- 📊 Evaluate performance on **binary** and **multiclass** prediction tasks
- ⚡ Provide an automated, configuration-driven experimentation pipeline
- 🖥️ Deploy an interactive dashboard for real-time prediction and analysis

## 📈 Dataset Summary

| Stage | Companies | Features |
| --- | --- | --- |
| Initial merged dataset | 86,114 | 24 |
| Cleaned financial dataset | 35,098 | 34 |
| Final multimodal dataset | 35,098 | 47 |

The trained models use **40 engineered features** (financial raw values, ratios, normalized scores, a financial-health score, and 13 NLP indicators).

### 📂 Data Sources

- **Primary**: SEC EDGAR XBRL financial filings (2022–2024, 12 quarters)
- **Secondary**: Public corporate credit rating datasets
- **Textual**: Synthetic MD&A-style disclosures (used to validate NLP pipeline)

> ⚠️ **Note**
>
> Raw SEC XBRL data (~41M records, ~6.6GB) and the large intermediate CSV files are
> intentionally **not included** in this repository due to size constraints. Only the
> final dataset the dashboard needs (`dashboard/credit_ratings_multimodal_final.csv`)
> is committed. All datasets can be regenerated by running the provided notebooks.

## 📋 Methodology

### 1. 📥 Data Extraction
- Parsed quarterly SEC XBRL numeric files (`num.txt`, `sub.txt`, `tag.txt`)
- Extracted key financial metrics (assets, liabilities, revenue, income, debt)
- Aggregated data at the company level

### 2. 🧹 Data Cleaning & Feature Engineering
- Missing value analysis and domain-aware imputation
- Outlier detection and removal
- Financial ratio computation (current ratio, debt-to-equity, ROA, profit margin)
- Derived features (normalized scores, financial health score, company size)

### 3. 📝 NLP Feature Engineering
- Synthetic MD&A-style text generation (for pipeline validation)
- Extraction of sentiment, risk, uncertainty, and readability metrics
- Correlation analysis between NLP features and credit ratings

### 4. 🤖 Machine Learning Modeling
- **Tasks**:
    - Binary: Investment Grade vs Non-Investment Grade
    - Multiclass: Credit rating categories (A, AA+, BBB, BB, B)
- **Models**: Random Forest, Gradient Boosting, Logistic Regression, SVM
- **Feature configurations**: Financial-only and Financial + NLP (multimodal)
- **Best model**: Gradient Boosting (used in the dashboard)

### 5. ⚙️ Automation & Reproducibility
- Modular Python scripts in `src/`
- YAML-based configuration
- Automated saving of metrics, results, and artifacts

## 📊 Results

All dashboard metrics below are **computed live from the trained models** on the held-out
test split (20%, stratified, `random_state=42`).

### Performance Improvement from Multimodal Features

- **Binary classification** 🎯 (Investment Grade)
    - Financial-only: 97.88%
    - Multimodal: 100.00%
    - Improvement: **+2.17%** 📈
- **Multiclass classification** 🏷️ (credit-rating category)
    - Financial-only: 93.76%
    - Multimodal: **95.94%**
    - Improvement: **+2.32%** 📈
    - Mean one-vs-rest AUC ≈ **0.998**

### Feature Importance (real, from the Gradient Boosting model)

Predictions are driven by a **mix of NLP and financial signals**:

| Rank | Feature | Importance |
| --- | --- | --- |
| 1 | `nlp_complexity` | ~23% |
| 2 | `current_ratio_norm` | ~18% |
| 3 | `current_ratio` | ~12% |
| 4 | `total_liabilities` | ~10% |
| 5 | `total_assets` | ~7% |

> ⚠️ **Note:** The very high binary accuracy reflects strong feature correlations in the
> dataset; potential leakage and time-aware validation are discussed in the report.

## 🚀 How to Run

### Option A — Run the interactive dashboard (quickest)

```bash
pip install -r requirements.txt
streamlit run dashboard/dashboard_app.py
```

Then open http://localhost:8501. The app loads the trained models and the final dataset
from the `dashboard/` folder.

### Option B — Reproduce the full research pipeline

The notebooks need the research environment (pandas, scikit-learn, nltk, etc.):

```bash
conda env create -f environment.yml
```

Then run the notebooks in order:

1. `01_data_extraction.ipynb` 🔍
2. `02_eda_preprocessing.ipynb` 🧹
3. `03_nlp_feature_engineering.ipynb` 📝
4. `04_ml_modeling.ipynb` 🤖
5. `05_pipeline_automation.ipynb` ⚡

Or run scripts directly:

```bash
python src/model_training.py
```

### ☁️ Deploying on Streamlit Cloud

Set **Python version = 3.13** in *Advanced settings* when deploying (Python 3.14 has no
prebuilt `scikit-learn==1.7.1` wheel and the build fails). Main file path:
`dashboard/dashboard_app.py`. Full instructions are in **`DEPLOYMENT.md`**.
